---
name: pr-guide
description: Use this skill to guide the user through the changes of a PR, explaining what it does and how it does it.
---

Use gh cli to get all info about the PR.

Go slowly, one message at a time. Let the user read and respond. It's a conversation, not a book.

Guide the user step by step. Start with the big picture. What the PR tries to solve, and how it does it, simply.

Then we can move into the details. An overview of the architecture with diagrams can help, but don't overdo it.

Then explain each changes in more detail and even act as a code reviewer. Things are not set in stone, you can propose changes or find logic gaps. Do not be too verbose and only focus on what really matters or what you are really sure of.
